# AWS S3 (Simple Storage Service) – Complete Beginner Guide

This **README.md** is a **complete beginner-friendly guide** to **AWS S3**. It explains **what S3 is, why it is used, all important concepts, and step-by-step usage** in very simple language.

---

## What is AWS S3? (Very Simple Explanation)

**Amazon S3 (Simple Storage Service)** is an **object storage service** used to store files such as:

- Images
- Videos
- Backups
- Logs
- Application data

Think of S3 as:

> A **highly durable online folder** where you store files securely.

---

## Why Do We Use S3?

We use S3 because it is:

- Highly durable (99.999999999%)
- Scalable (store unlimited data)
- Secure
- Cost-effective
- Accessible from anywhere

---

## Key S3 Concepts (Must Know)

### 1. Bucket

A **bucket** is a container for objects.

- Bucket name must be **globally unique**
- Example: `my-app-backups`

### 2. Object

An **object** is a file stored in S3.

- Object = File + Metadata

### 3. Key

The **key** is the file path inside a bucket.

- Example: `images/photo.jpg`

### 4. Region

Buckets are created in a **specific AWS region**.

---

## S3 Storage Classes

Different storage classes are used for different purposes.

| Storage Class              | Use Case                 |
| -------------------------- | ------------------------ |
| Standard                   | Frequently accessed data |
| Intelligent-Tiering        | Unknown access pattern   |
| Standard-IA                | Infrequent access        |
| One Zone-IA                | Lower cost, single AZ    |
| Glacier Instant Retrieval  | Archive, fast access     |
| Glacier Flexible Retrieval | Archive, slower          |
| Glacier Deep Archive       | Long-term archive        |

---

## S3 Use Cases

- Static website hosting
- Application file storage
- Backup and restore
- Log storage

# STEP 1: Create an S3 Bucket

1. Login to AWS Console
2. Search for **S3**
3. Click **Create bucket**
4. Enter:
   - Bucket name: `my-first-s3-bucket`
   - Region: Choose nearest region

5. Leave other settings default
6. Click **Create bucket**

---

# STEP 2: Upload Objects (Files)

1. Open your bucket
2. Click **Upload**
3. Add files
4. Click **Upload**

Files are now stored in S3.

---

# STEP 3: Access Control & Permissions

S3 is **private by default**.

### Ways to Control Access

1. Bucket Policy
2. IAM Policy
3. ACL (Not recommended)

---

## Example: Make Object Public (Not Recommended for Production)

1. Open the object
2. Go to **Permissions**
3. Disable "Block public access"
4. Allow public read

---

# STEP 4: S3 Bucket Policy (Basic Example)

A bucket policy controls who can access the bucket.

Example (Read-only public access):

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::my-first-s3-bucket/*"
    }
  ]
}
```

---

# STEP 5: Enable Versioning

Versioning keeps multiple versions of objects.

1. Open bucket
2. Go to **Properties**
3. Enable **Versioning**

Benefits:

- Protects from accidental deletion
- Easy rollback

---

# STEP 6: Enable Encryption

Encryption protects data at rest.

### Types of Encryption

- SSE-S3 (AWS managed)
- SSE-KMS (KMS managed)
- SSE-C (Customer managed)

Recommended:

- **SSE-S3 or SSE-KMS**

---

# STEP 7: Lifecycle Rules

Lifecycle rules automatically move or delete data.

Example:

- Move to Glacier after 30 days
- Delete after 365 days

Steps:

1. Go to **Management** tab
2. Create lifecycle rule

---

# STEP 8: Static Website Hosting

S3 can host static websites.

1. Go to **Properties**
2. Enable **Static website hosting**
3. Upload `index.html`

---

# STEP 9: S3 Security Best Practices

- Block public access
- Use IAM roles
- Enable versioning
- Enable encryption
- Enable logging

---

# STEP 10: S3 Pricing Basics

You pay for:

- Storage used

- Data transfer

Lower cost by:

## 1. S3 + CloudFront

### What is CloudFront?

CloudFront is a **Content Delivery Network (CDN)** that delivers content faster using **edge locations** across the world.

### Why use S3 with CloudFront?

- Faster content delivery
- Lower latency
- Better security
- Reduced load on S3

### Common Use Case

- Hosting static websites
- Serving images, videos, JS, CSS files

---

### Architecture (Simple)

User → CloudFront → S3 Bucket

---

### Step-by-Step: S3 + CloudFront

#### Step 1: Create an S3 Bucket

- Create a bucket
- Upload static files (HTML, CSS, images)
- Keep bucket **private** (recommended)

#### Step 2: Create a CloudFront Distribution

1. Go to CloudFront
2. Click **Create Distribution**
3. Origin Domain → Select your S3 bucket
4. Enable **Origin Access Control (OAC)** (recommended)
5. Default settings → Create distribution

#### Step 3: Update S3 Bucket Policy

Allow CloudFront to access S3 (not public users).

#### Step 4: Access Website

- Copy CloudFront **Distribution Domain Name**
- Open it in browser

✅ Your content is now served faster and securely.

---

## 2. S3 + IAM Roles

### What is an IAM Role?

An IAM role allows AWS services (like EC2) to **access S3 securely** without using access keys.

### Why use IAM Roles with S3?

- More secure than access keys
- Easy permission management
- Recommended AWS best practice

---

### Use Case

EC2 instance uploads files to S3 automatically.

---

### Step-by-Step: S3 + IAM Role

#### Step 1: Create IAM Role

1. Go to IAM → Roles → Create role
2. Trusted entity → EC2
3. Attach policy:
   - `AmazonS3FullAccess` (for learning)
4. Create role

#### Step 2: Attach Role to EC2

1. Go to EC2
2. Select instance
3. Actions → Security → Modify IAM Role
4. Attach created role

#### Step 3: Test Access from EC2

```bash
aws s3 ls
aws s3 cp test.txt s3://your-bucket-name/
```
